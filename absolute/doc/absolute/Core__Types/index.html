Module Core__Types

We represent the abstract and concrete types of variables. Design rational: We need to distinguish between concrete and abstract types because `Real` can be "instantiated" to a float or rational number. Rational are more precise than float but in essence they aim to represent the same solutions space. Note: It is not in `Lang` to avoid circular dependencies between `Bounds` and `Lang`. Indeed, we need to associate an abstract type to each bound.

type value_ty =
| Z
| Q
| F

value_ty symbolically represents integers (Z), rational (Q), and floating point numbers (F). It is also used in Lang.Typing.Tast.

type var_abstract_ty =
| VUnit
| Bool
| Machine of value_ty

Machine representation of value_ty.

| BDD of int

`Integer` is considered to be over 32 bits.

type var_concrete_ty =
| Int
| Real
type var_ty =
| Concrete of var_concrete_ty
| Abstract of var_abstract_ty
